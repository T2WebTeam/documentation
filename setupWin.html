Installation Procedure

  1. Download apache and extract it
  2. Move the apache24 folder to your C: drive.
  3. After download & extract php, open 'php7' folder in Download, move 'php' folder to C: drive.
  4. Download Microsoft visual C++. Download only the file that has the same bit for the laptop.
    • 64 bit = x64
    • 32 bit = x86
  5. After downloading Microsoft Visual C++ , double click on it to do the installation.
  6. Launch CMD as administrator
  7. Type “C:\apache24\bin\httpd -k install” . ( If there is error mentioning you do not have any specific dll file , download microsoft visual C++ 2012/2015 should be able to solve it. )
  8. Right click on the Window Button and open System.
  9. Go “Advanced System Setting” > “Advance” Tab > “Environment Valuables”
  10. Go to “Path” Under “System Variable”.
  11. Add these new variable /file path
    • C:\php
    • C:\Apache24\
    • C:\Apache24\bin
  12. Go to C:\Apache24\bin
  13. Create shortcut for Apache Monitor and move to desktop.
  14. Restart laptop.
  15. Open Apache Monitor and make sure apache starts automatically.

Setup Project

Clone Repository

  1. Create an account using company email

    https://gitlab.ttwoweb.com/

  2. Get the project link from senior

  3. Copy url in 'Clone with HTTP'

  4. Open sublime merge
  5. Go to File > Clone Repository
    1. Paste the url
    2. Make sure the path is C:\Apache24\htdocs\PROJECTNAME
    3. Click 'Clone'
  6. Go to Repository > Create Branch From Remote Branch
    1. Click origin/test
    2. Repeat and click origin/dev (if got)

  7. Make sure your current branch is 'TEST'

Setup Server IP Address
  1. Open Notepad++
  2. In Notepad++, open hosts file in C:\Windows\System32\drivers\etc

  3. Add server IP address after last line

    103.76.36.101 testfront2 103.76.36.134 newtestfront3

  4. Add project IP address after last line

    ## PROJECTNAME 127.0.0.1 PROJECTNAME.member 127.0.0.1 PROJECTNAME.admin xxxxxxxxx testbackend.PROJECTNAME.testbackx //depends on project, can get from senior 103.76.36.101 testPROJECTNAMEsuperadmin.speed101.pw
  5. Save the file
    1. Will ask permission to edit as administrator, click yes
    2. Save again
  6. Restart Apache

*You need to restart Apache everytime you edit hosts file

Setup Local URL (vhosts.conf)
  1. Open Notepad++
  2. In Notepad++, open vhosts.conf file in C:\Apache24\conf

  3. Add your project directory path and local url

    <VirtualHost *:80> DocumentRoot "C:\Apache24\htdocs\PROJECTNAMEFrontend\admin" ServerName PROJECTNAME.admin ErrorLog "logs/PROJECTNAME.admin-error.log" CustomLog "logs/PROJECTNAME.admin.log" common </VirtualHost>
    <VirtualHost *:80> DocumentRoot "C:\Apache24\htdocs\PROJECTNAMEFrontend\member" ServerName PROJECTNAME.member ErrorLog "logs/PROJECTNAME.member-error.log" CustomLog "logs/PROJECTNAME.member.log" common </VirtualHost>
  4. Save the file
  5. Restart Apache

*You need to restart Apache everytime you edit vhosts.conf file

After done setup host and vhosts.conf, open your browser and check whether you can access the web page

Commiting Code

  1. Make sure you commit your code in test branch
  2. Pull before you commit your code

  3. Write commit message based on the format below

    • Module/Feature

      [Admin][Module][pageName] description [Member][Module][pageName] description
      eg. [Member][NicePay][payment.php] plugin nicepay module eg. [Admin][NicePay][paymentListing.php] plugin nicepay payment listing

    • Document Tuning

      [Admin][doc + date][pageName] description [Member][doc + date][pageName] description
      eg. [Admin][doc140622][portfolio.php] add 'memberID' column & filter

    • Normal Tuning

      [Admin][pageName] description [Member][pageName] description
      eg. [Member][support.php] tune input background color

    • If commited a commit but still need to commit another tuning

      Add (No) behind the description
      eg. [Member][profile.php] tune client ID to member ID eg. [Member][profile.php] tune client ID to member ID (2)
    • For document tuning which involve many pages, if not related to layout, please separate the commit.
    • For document tuning which related to layout or one whole set, can commit together.
  4. Commit and push your commit to test branch
